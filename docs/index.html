交互式网络排查 CLINode.js跨平台

netq

面向开发/运维/普通用户的轻量网络排查工具。核心目标只有一个：把常用网络诊断变成“看得懂的一键结果”，而不是让你背一堆命令参数。

查看 README GitHub 仓库

交互模式

菜单式选择，无需记命令；适合临时排查。

非交互模式

一条命令输出结果；适合脚本/CI。

导出报告

结果可导出 TXT/JSON 到本机目录。

最常用的 4 条命令

netq
netq --doctor github.com --ports "80,443,3000-3010"
netq --tcp github.com 443
netq --http https://github.com
设计原则能交互就交互，能一键就一键；输出优先可读，其次可机器读（JSON）。
无彩色输出--no-color 或环境变量 NO_COLOR=1

功能一览

公网 IP快速获取对外出口 IP（适合排查 NAT / 代理 / 出口变化）。
DNSlookup + A/AAAA 基本记录查询；必要时可拓展更多类型。
Ping / Traceroute连通性与路径定位，快速判断“通不通、卡在哪”。
TCP / HTTP端口连通性、HTTP 状态码/重定向链，辅助判断服务是否可达。
监听端口本机 Listening 列表，Windows 可解析 PID 进程名。

非交互（JSON）示例

# 机器可读输出（失败会退出码=1）
netq --doctor-json github.com --ports "80,443"

# 输出结构大致如下：
{
  "command": "doctor",
  "timestamp": "2026-03-17T00:00:00.000Z",
  "host": "github.com",
  "ports": [80, 443],
  "checks": { },
  "ok": true
}

提示：脚本中可直接用退出码判断成败；或解析 ok 字段。
